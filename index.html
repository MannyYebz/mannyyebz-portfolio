Options Pricing Library

A modular Python library implementing three independent pricing models — each documented, independently validated, and structured for reusability and transparency.

What I Built

  • Black-Scholes Model — Closed-form analytical pricing for European options with a full Greeks suite (Delta, Gamma, Theta, Vega, Rho) quantifying sensitivity to every market variable.
  • Monte Carlo Simulation — Stochastic price path generation via Geometric Brownian Motion across thousands of scenarios, producing convergent estimates with statistical confidence intervals.
  • Binomial Tree Model — Discrete-time lattice pricing supporting American-style options with early exercise logic, configurable step count, and backward induction valuation.

Why It Matters for AI & Data

This project demonstrates structured, well-documented code in a domain where correctness is non-negotiable. Each module is independently validated and designed so outputs are interpretable — the same standard required in high-quality training datasets and AI evaluation pipelines. Building models others can read and trust is a data skill, not just a finance one.

R · Credit Risk · Dataset Modeling View on GitHub

Credit Risk & LGD Model

A probability-weighted credit analysis framework in R, applied across 15+ healthcare REITs to model Loss Given Default and recovery under multiple distress scenarios.

What I Did

  • Assembled and structured a custom dataset covering 15+ healthcare REIT issuers — cleaning, normalizing, and labeling financial variables for consistency across the full sample.
  • Built probability-weighted recovery models assigning scenario weights to distress outcomes (restructuring, liquidation, par recovery) based on issuer-specific risk factors.
  • Applied collateral coverage analysis and recovery rate benchmarking to stress-test LGD estimates against historical comps and sector-level default data.

Why It Matters for AI & Data

The core challenge here was data, not computation. Deciding how to label, categorize, and weight each issuer's distress scenario required deliberate annotation logic — the same skill set that drives effective data labeling and quality control in AI training pipelines. This project demonstrates how to impose structure on messy, real-world data and produce outputs that are defensible and reproducible.

03
Python / R · DCF · Monte Carlo View on GitHub

Healthcare Real Estate DCF & Monte Carlo Model

A full acquisition underwriting model for a $25M healthcare real estate target — integrating DCF analysis, Monte Carlo simulation, and scenario-based evaluation into a single auditable framework.

What I Built

  • DCF Engine — 10-year free cash flow projections using healthcare-specific operating assumptions (occupancy, rental escalations, capex cycles), discounted at a risk-adjusted WACC to produce baseline intrinsic value.
  • Monte Carlo Layer — Stochastic variation applied to key inputs (cap rates, NOI growth, vacancy) across 10,000+ simulated acquisitions, producing probability distributions over IRR and equity multiple outcomes.
  • Scenario Framework — Bull / base / bear scenario matrix with labeled assumption sets, enabling direct comparison of outcome distributions and downside risk quantification.

Why It Matters for AI & Data

This model is structured end-to-end for auditability. Every assumption is labeled, every scenario is named, and the output distributions are designed to communicate uncertainty, not obscure it. In AI data contexts, this is exactly the mindset needed to build evaluation sets, annotate edge cases, and produce documentation that makes model behavior interpretable to both technical and non-technical stakeholders.
